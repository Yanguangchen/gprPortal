## Overview
A visual redesign of the **Upload GPR Image** screen of the GPR Portal (the RAK Materials radar image management system). Same brand DNA — Ocean glassmorphism, cyan accent, Outfit font, animated gradient + orb backdrop — but reorganized for better hierarchy and less wasted space.

## About the design files
`GPRPortal Upload.html` in this folder is a **design reference built in plain HTML/CSS/JS** — it shows the intended look and behavior. It is **not** meant to be shipped as-is. Recreate it inside this repo's existing structure:
- Markup is generated by ES-module classes in `js/` (e.g. `js/UploadPanel.js`), not static HTML.
- All styling lives in `style.css` and is driven by the per-theme CSS custom properties under `[data-theme="…"]`.
- Keep using those modules and tokens. Do **not** introduce a framework, inline styles, or a CSS-in-JS lib.

## Fidelity
**High-fidelity.** Colors, spacing, typography, and interactions in the reference are final. Recreate pixel-faithfully using the existing token system.

## What changes (mapped to this repo's files)

The redesign merges them into a **single** slim bar containing: brand mark+name, a vertical divider, the system descriptor, then right-aligned: audio toggle, theme switcher, user chip.

- Remove the standalone `#user-profile` card from the page flow; fold the avatar + name + email into a compact **user chip** on the right of the header.
- Keep the existing mount points (`#audio-switcher`, `#theme-switcher`) — just relocate them into the new top-bar actions row.
- The brand tagline changes from "Specifically made for RAK Materials PTE LTD" to a compact eyebrow: **`RAK Materials Pte Ltd`** under the wordmark.
- Add a new **page heading** block between the top bar and the upload card:
  - `<h1>` "Upload GPR Image" with a 9px glowing accent dot before it.
  - Sub: "Add a new radar scan to the portal. Tag it with project metadata so your team can find it later."
  - Right-aligned breadcrumb: **Portal / Upload / New scan** (the word "Portal" in accent color).

## Interactions & behavior
- **Theme switch** — segmented control sets `data-theme` on `<html>`; active segment gets `--accent-soft` bg. (Reuse existing `js/theme.js`.)
- **Drag & drop** — `dragenter/over` add `.over` (accent border + 4px glow ring); drop or browse loads a preview `<img>` filling the zone with a bottom **preview bar** (filename + size + remove X). Remove restores the empty state.
- **Radar sweep** — `.sweep` conic-gradient rotates 4s linear infinite (intensifies on hover). Disable under high-contrast / `prefers-reduced-motion`.
- **Validation** — on submit, check file -> company -> project -> site -> date in order; first failure sets `.status.error` with an alert icon and focuses the field. Missing file also flashes the drop zone.
- **Upload progress** — show `.prog` bar animating 0->100%, button shows spinner + "Uploading...", then reset + success status + toast. (Wire to the real `api.create` `onProgress` callback that already exists in `UploadPanel`.)
- **Easing** — `cubic-bezier(.4,0,.2,1)`, ~.22s. Toast in: `cubic-bezier(.16,1,.3,1)`, .45s.

## Assets
- **Icons**: all inline stroked SVG (1.7-2.0 stroke, round caps), Lucide-style. No icon font needed — keep them inline in the generated markup, consistent with the current codebase.
- **Avatar**: reference uses a "CY" initials SVG placeholder. Use the real Google profile photo from your auth flow (the repo already wires `user-avatar`).
- **Logo mark**: reference draws a hexagon + radar dish/ping in SVG. You can keep the existing hex glyph or adopt the SVG mark from the reference — your call.
